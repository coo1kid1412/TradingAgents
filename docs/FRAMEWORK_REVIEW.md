# TradingAgents 全链路框架评审（金融视角）

> 撰写视角：以卖方研究 + 买方 PM + 风控官的真实工作流为参照，评估当前 11 个 agent 的人设、CoT、I/O 与链路衔接。本文不是单项优化 spec，是后续路线图重排的依据。

---

## 一、执行摘要

### 当前框架的 5 个底层问题

1. **人设过浅**：多数 agent 是"贴标签式"角色（"你是一名 X"），缺少职业训练、思维习惯、决策框架。LLM 拿到这种 prompt，行为退化为"通用智能体在某个标签下扮演"
2. **CoT 没强结构**：bull/bear 没有论点-证据-反证-定论的 thesis 模板；fundamentals 缺 DuPont、可比公司；market 缺市场制度（regime）判断；导致输出风格化但深度不足
3. **风控辩论是伪职能**："激进/保守/中立"是风险偏好分类，不是职能分类。真实风控按"宏观-头寸-尾部-相关性"分工。当前三方常常退化成 bull/bear 第二轮辩论
4. **链路冗余 + 关键数据缺失**：每个下游 agent 读所有上游报告（context 膨胀），但 trader 看不到原始量价、风控看不到具体头寸/VaR 数字、PM 看不到上次同票的历史决策
5. **输出无标准化决策卡**：portfolio_manager 的 decision.md 没有"评级｜目标价｜时间窗｜仓位｜核心催化｜核心风险"这种 1 页摘要。用户必须读 200-400 行才能拿到关键决策点

### 推荐的下一阶段路线（替换原 04/05）

| 序号 | 主题 | 价值 | 量级 |
|------|------|------|------|
| 03 | 敏感性自检算术统一（已规格） | 修补技术债 | 单文件 |
| 04 | **决策卡（Decision Card）标准化** | 解决"看 5_portfolio 够不够" | 单文件 |
| 05 | **入场时机决策模块** + 分析师权重显式化 | 解决 HOLD 居多痛点 | 双文件 |
| 06 | **fundamentals 引入 DuPont + 可比公司框架** | 提升基本面深度 | 单文件 |
| 07 | **市场制度（regime）识别前置** | market_analyst 升级 | 单文件 |
| 08 | **风控辩论职能化重构** | 把"激进/保守/中立"改成"宏观/头寸/尾部" | 三文件 |
| 09 | **bull/bear thesis 模板化** | 强制 thesis-evidence-killer-test 结构 | 双文件 |

后续 06-09 是较大的架构重构，不必全做，按你判断价值与成本择优。

## 二、角色级评估

### 2.1 Market Analyst（市场分析师）

**现状人设**：从指标目录挑 ≤8 个指标 → 写技术分析报告 → 附表

**输入**：股票代码 + 当前日期；预取 365 天 OHLCV；可调 `get_indicators` 取技术指标

**输出**：`market_report`（自由格式 markdown）

**核心问题**：
- **缺市场制度（regime）判断**。真实技术分析师第一步永远是判 regime（趋势/震荡/突破/趋势末期），然后才选指标——RSI 在震荡有效、在强趋势中失效；MACD 在趋势中有效、在震荡中假信号多
- **指标选择缺纪律**。当前是让 LLM 自由"挑 8 个互补指标"，但没有规定每种 regime 下该挑什么指标组合。结果是不同标的的指标组合差异大、可比性差
- **量价关系被弱化**。VWMA 是清单里唯一带 volume 的指标。但真实技术分析里"量价配合"是核心——放量突破 vs 缩量阴跌，意义截然不同。当前没有 volume profile、累积/派发线（A/D Line）、OBV
- **无支撑/阻力位的明示输出**。分析里偶尔提及，但没有结构化字段。这是 trader 最需要的输入

**改造建议**：
1. CoT 强制三步：① 判 regime（4 选 1）② 按 regime 选指标组合（每种 regime 给定推荐组合）③ 输出标准化字段：当前价 / 关键支撑位 / 关键阻力位 / regime / RSI / MACD 状态 / ATR
2. 增加 volume 维度的强制描述："最近 5/20 日量比，主力资金近 5 日累计净流入/流出"
3. 输出附结构化"技术面快照表"，下游可机读

---

### 2.2 Fundamentals Analyst（基本面分析师）

**现状人设**：拉取 fundamentals/balance/cashflow/income，按 8 节结构写报告（公司概况/估值/盈利/偿债/成长/现金流/建议/汇总表）

**输入**：股票代码 + 当前日期；通过 `_fetch_all_fundamentals` 并行拉数据

**输出**：`fundamentals_report` + 关键指标汇总表

**核心问题**：
- **没有 DuPont 分解**。ROE = 净利率 × 资产周转 × 杠杆。这是判断"盈利质量来自哪里"的标准方法。当前只罗列 ROE 数值，不分解
- **没有可比公司（peer comparison）框架**。"PE 35 倍贵"在没有同业对比时是空话。真实基本面分析必须给出 PE/PB/PS/EV-EBITDA 的同业分位数
- **没有 ROIC vs WACC 的资本效率判断**。增长是"创造价值"还是"摧毁资本"取决于 ROIC > WACC
- **盈利质量分析浅**。当前看现金流/净利润比例，但没看应收账款变化率、存货变化率、预收账款 → 这些是判断"账面利润是否真实兑现"的关键
- **管理层资本配置评分缺失**。回购 vs 分红 vs 再投资 vs 收购的历史轨迹完全未涉及

**改造建议**：
1. 在第三节"盈利能力"加 DuPont 分解段落：写明 ROE = 净利率 × 周转 × 杠杆，标出哪一项是主驱动 / 哪一项是隐忧
2. 新增"同业对标"小节：基于行业代码或主营业务描述，要求 LLM 生成 3 家可比公司及其 PE/PB/PS 中位数，标注当前公司的分位数
3. 在"现金流分析"节加"盈利质量校验三件套"：应收周转率、存货周转率、经营现金流/净利润比例。每项给出"健康/警示/危险"标签
4. 新增"资本效率"小节：估算 ROIC（NOPAT/投入资本），与 WACC（用行业平均 8-10% 近似）对比

### 2.3 News Analyst（新闻分析师）

**现状人设**：调用多个新闻工具 → 按公司/行业/宏观分类 → 标注可信度 → 输出汇总表

**输入**：股票代码 + 当前日期；A 股可用 7 个工具，港美股 4 个

**输出**：`news_report` + 汇总表（来源/日期/分类/内容/可信度/股价影响）

**核心问题**：
- **缺催化剂（catalyst）类型分类**。新闻不只是分公司/行业/宏观，还得分 catalyst type：earnings / M&A / 监管 / 管理层 / 产品 / 诉讼 / 大宗交易。每种 catalyst 的市场反应不同
- **缺时间窗口标注**。一条新闻是"立即兑现"还是"30 天/90 天/更长才发酵"？没有时间标签，下游无法判断这条新闻对当前介入决策的边际影响
- **方向性标签太弱**。当前只标"对股价影响"自由文本，没有结构化的 +/-/中性 + 强度
- **未排序信息密度**。10 条新闻一起列出，没区分"决定性 1 条"和"背景 9 条"。读者很难抓重点

**改造建议**：
1. 汇总表升级为 7 列：来源｜日期｜**催化剂类型**｜分类｜**时间窗（即时/30D/90D/长期）**｜方向（+++/++/+/0/-/--/---）｜可信度
2. 在表前强制"Top 3 关键新闻"小节：报告里只挑 3 条最重要的逐条点评，其他放表里
3. 引入"新闻一致性"判断：多源是否一致？分歧在哪？这影响下游 LLM 评分时的可信度

---

### 2.4 Social Media Analyst（舆情分析师）

**现状人设**：通过 Xueqiu + 财经新闻 → 评估整体情绪 → 标注关键叙事 / 情绪转向 / 风险信号

**输入**：股票代码 + 当前日期；Xueqiu 帖子搜索（最多 3 次）

**输出**：`sentiment_report` + 汇总表

**核心问题**（这是当前框架最弱的一环）：
- **散户情绪在中国 A 股上是反向指标更多**。Xueqiu 高热度 + 一致看多，往往对应短期顶部。当前的 prompt 让 LLM"判断整体情绪"，但没有让它做反向解读
- **真正的"市场情绪"信号被忽略了**：龙虎榜、北向资金、融资余额变化、ETF 资金流向、券商研报评级变化频率——这些是机构资金行为，远比散户帖子有信号
- **大 V 影响力评估难有效**。LLM 不知道哪个账号在投资圈真的有影响力
- **报告价值低**。从 5 份历史报告看，sentiment_report 几乎从来不影响最终评级

**改造建议**（最激进的可能是退化，最保守是改进）：
1. **保守改进**：把 Xueqiu 数据当反向指标看待——"如果 Xueqiu 极度看多 + 帖子量激增，输出'警惕散户接盘'信号"
2. **较激进**：新增机构行为数据源：北向资金/龙虎榜/融资余额（A 股可得），把"舆情分析师"重命名为"资金面分析师"，主输入从 Xueqiu 改为机构资金数据
3. **最激进**：直接退役 sentiment_analyst，把舆情作为一个"附录"放在 news_analyst 末尾。把节省下来的 LLM 调用配额给基本面或市场分析师做更深的分析

我个人推荐 **方案 2**——保留位置但换数据源和定位，从"散户情绪"改为"机构资金行为"。这才是真正有信号的"市场情绪"。

### 2.5 Bull / Bear Researcher（多/空研究员）

**现状人设**：bull 找有利论据反驳 bear，bear 反之。各自参考四份分析师报告 + 历史教训

**输入**：四份分析师报告 + 辩论历史 + 对手最新发言 + 历史教训

**输出**：累加到 `investment_debate_state.history` 的发言

**核心问题**：
- **没有 thesis 模板**。专业 long/short 报告必须有：① 投资论点（1 句话）② 关键驱动（3-5 个具体催化剂）③ killer test（什么会让我改主意）④ 时间窗 ⑤ 主要风险。当前 bull/bear 输出是"对话式"自由发挥，结构松散
- **倾向于扩列论据数量而非加深论据强度**。LLM 当 bull 时，会"我能找到 8 个看多理由！"——但其中 5 个可能是弱论据。这给下游 RM 评分造成困扰（10 弱论据 vs 5 强论据怎么比？）
- **辩论像 debate club，不像 investment committee**。"Output conversationally as if you are speaking" 这条 prompt 让输出口语化，但牺牲了证据-论证的严谨度
- **历史教训注入是无差别**。bull 会被注入对所有方向的过去教训。但一个看空的过去教训对当前看多论证可能反而干扰

**改造建议**：
1. 强制 thesis 模板，输出固定 5 节：核心论点 / 关键驱动 / killer test / 时间窗 / 主要风险
2. 引入论据强度自评：每条论据自评 1-3 星（弱/中/强），下游 RM 评分时按强度加权
3. 历史教训按方向过滤：bull 只看历史"看多正确/错误"教训，bear 只看历史"看空正确/错误"教训
4. 把"Output conversationally"改成"Output structured（论点-证据-推理-结论）"

### 2.6 Research Manager（研究经理）

**现状人设**：综合 bull/bear → 评分 → 阈值映射到 5 档评级 → 敏感性自检 → 投资计划

**输入**：四份分析师报告 + bull/bear 辩论历史 + 历史教训

**输出**：`investment_plan`（含评级、行动方案）+ 评分表

**核心问题**：
- **评分缺加权机制**。"市场分析最高、基本面中高、新闻中、舆论中低"这种你直觉认为合理的权重，没有写进 prompt。当前 RM 给所有论据等权
- **没区分"看法"和"时机"**。RM 评级是"看多/看空"，但没回答"现在介入合不合适"。BUY 评级 + 高 PE 滞涨 = 别现在追，等回调——这种判断当前框架不存在
- **历史教训利用浅**。memory.get_memories 用的是 cosine similarity，但没有"按当前评级方向 / 行业 / 技术形态"做二级过滤
- **判官身份与辩论参与者身份混淆**。RM 既要打分又要出"投资计划"——前者要求中立，后者要求决断。一个 prompt 里压两个职责

**改造建议**：
1. 在第一步评分之前，加"分析师权重声明"：明确告诉 RM "市场分析的论据权重 30%、基本面 30%、新闻 25%、舆论 15%"（具体数字可调）
2. 评级输出后，独立加"入场时机判断"小节：评级（看多/看空）+ 时机（立即/等回调/等突破/不入），二维独立
3. 把 RM 拆成两段（不必拆 agent，prompt 里分两段）：先做"裁判"打分得评级，再做"PM 助理"出投资计划

### 2.7 Trader（交易员，已优化为执行专家）

**现状人设**：采纳 RM 方向 → 输出执行方案（入场/止损/流动性/执行风险）

**输入**：investment_plan + 四份分析师报告 + 历史教训

**输出**：`trader_investment_plan`

**核心问题**（优化 01 后已大幅改善，但仍有）：
- **没接触原始市场数据**。trader 工作在"分析师报告的二手转述"上。它说"日均成交额 5000 万"是猜的，因为没看到 OHLCV 原始数据
- **优化 01 的 ETF 副产物**：159326 trader.md 里"需查询近期数据"占位文本——这是数据没注入的具体表现

**改造建议**：
1. 给 trader 节点的 state 注入额外字段：近 20 日日均成交额、近 5 日 ATR、近期最大单日跌幅。这些数据 market_analyst 已经拉过，存到 state 里复用即可
2. 让 trader 在"流动性评估"小节直接引用注入的数字，不再"估计"

### 2.8 Risk Debaters（激进/保守/中立，三方辩论）

**现状人设**：分别从激进/保守/中立角度反驳另外两方

**输入**：trader_plan + 四份分析师报告 + 辩论历史 + 对方最新发言

**输出**：累加到 `risk_debate_state` 的辩论历史

**核心问题**（**这一组是当前框架最大架构债**）：
- **风险偏好 ≠ 风险职能**。激进/保守/中立是态度光谱，不是分工。真实风控团队按职能分工：宏观/制度风险、头寸/相关性风险、尾部风险、操作/合规风险
- **三方辩论容易退化为 bull/bear 第二轮**。激进 = bull++；保守 = bear++；中立 = RM 复述。我读历史报告时反复看到这个模式。优化 01-02 验收时也观察到
- **缺乏定量约束**。三方都在说"风险大"或"机会大"，但没有 VaR、最大回撤情景、相关性等具体数字
- **没有人在做"组合层"风险**。如果用户已持仓 9 只 AI 板块股票，再加这只协创数据组合风险已经爆了——但当前框架根本不知道"用户的现有组合"

**改造建议（中长期，工作量较大）**：
彻底重构三方为职能化：
- **宏观/制度分析师**：当前市场处于什么 regime（牛/熊/震荡）？该 regime 下此评级的胜率？
- **头寸/相关性分析师**：建议仓位上限基于波动率倒数 + 与现有资产的相关性
- **尾部风险分析师**：极端情况下最大回撤 / 停牌 / 跌停连续板的概率与应对

这样三方互补、不互斗，PM 接到的是三个维度的独立评估而非三方吵架。

### 2.9 Portfolio Manager（投资组合经理）

**现状人设**：综合风控辩论 → 默认采纳 RM 评级，可 ±1 档调整 → 出最终决策

**输入**：研究计划 + trader 执行方案 + 风控辩论 + 四份分析师报告 + 历史教训（含种子教训）

**输出**：`final_trade_decision`（decision.md）

**核心问题**：
- **decision.md 没有"决策卡"摘要**。读者必须读 200-400 行才能拿到结论。专业研报第一页是 1 张卡：评级 / 目标价 / 时间窗 / 仓位 / 核心催化 / 核心风险
- **不接触历史决策**。如果同一标的之前跑过，PM 不知道"我上次说什么、当时基础是什么、现在变了什么"
- **执行可行性核查（优化 01 加的）和方向决策没有形式化整合**。PM 可以在执行风险大时降一档，但没有"执行风险评分 → 评级调整量"的具体映射
- **种子教训只注入 PM 是好设计，但教训质量参差不齐**。`seed_lessons.py` 里有些教训具体（"权重股拉升掩盖个股下跌"），有些抽象（"保持谦卑"），LLM 适用判断成本高

**改造建议**：
1. **强制决策卡**（最高优先）：decision.md 头部固定一张表，9 个字段必填：评级 / 评级置信度 / 目标价 / 止损位 / 时间窗 / 建议仓位 / 入场时机（立即/等回调/等突破/不入）/ 核心催化（最多 3 条）/ 核心风险（最多 3 条）
2. **加"上次决策对照"小节**（如果之前跑过同标的）：上次评级 / 当时关键假设 / 假设是否仍成立 / 评级是否需要调整
3. **种子教训加适用性判断**：每条种子教训打"适用场景标签"，LLM 在引用前先判断匹配度

## 三、链路级问题

### 3.1 Context 膨胀

每个下游 agent 读所有上游报告，到 PM 时累计 3000+ 行 context。LLM 注意力被稀释，关键信息容易被淹没。

**改造方向**：
- 每个 agent 的输出都附"摘要 / 全文"双格式。下游优先读摘要，必要时引全文
- bull/bear 辩论历史在传给 RM 时，让 LLM 先做一轮压缩（≤500 字）

### 3.2 数据流缺口

| 数据 | 谁需要 | 谁拿到了 | 缺口 |
|------|--------|---------|------|
| 原始 OHLCV / 量比 / ATR | trader | market_analyst 拿到了，但没传到 state | trader 看不到原始量价 |
| 北向资金 / 融资余额 / 龙虎榜 | sentiment（应该）/ PM | 当前未接入 | 机构资金信号缺失 |
| 同业可比公司估值 | fundamentals | 未接入 | 估值无横向对照 |
| 用户现有组合 | PM | 当前架构没有"用户上下文" | 单标的独立分析，无组合维度 |
| 上次同标的决策 | PM | memory 通过 similarity 找，命中率不稳 | 缺确定性的"历史决策"通道 |

### 3.3 反思（reflect）机制的盲点

`reflect_and_remember(returns_losses)` 由用户回灌"实际收益"才触发学习。但用户大多数情况只看一次决策、不会回灌。这意味着 reflection 几乎不发生，memory 长期处于"种子教训 + 早期记录"的静态状态。

**改造方向**：让 reflect 不只依赖"实际收益"，也可以由"决策一致性"触发——例如同一标的连续 3 次跑出 HOLD，自动写入"该标的处于不可决判区"教训。

### 3.4 评级体系本身的张力

5 档评级（BUY/OVERWEIGHT/HOLD/UNDERWEIGHT/SELL）+ 敏感性自检 = 强制把脆弱的方向信号推向 HOLD。这是设计意图，但代价是用户痛点 1（HOLD 居多）。

**真正的解法**：把"评级"和"入场时机"二维独立。BUY 评级 + 不建议立即介入 = 用户能听懂的建议。优化 04（决策卡）和 05（入场时机）就是冲这个去。
