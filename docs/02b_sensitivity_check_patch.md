# 优化 02b：敏感性自检的边界包含 + 压力测试方法论补丁

## 1. 背景

优化 02 实施后跑批暴露了 2 个 spec 漏洞，`02_rating_threshold_relaxation_验收报告.md` 已记录。本文档是针对这两个漏洞的最小补丁，**不引入新功能**。

### 1.1 漏洞 A：边界包含关系不清晰

证据 —— `reports/159326_电网设备ETF华夏_20260509_230133/2_research/manager.md` 第 70 行：

```
| 是否跨越档位边界 | **否**（-1 仍落在 SELL 区间 d < -1） |
```

LLM 把 d=-1 归到 SELL 区间，但优化 02 的 spec 表格写的是：

```
| -1 ≤ d ≤ -0.5 | UNDERWEIGHT |
| d < -1        | SELL        |
```

按规格 d=-1 是 UNDERWEIGHT 的左端点，应触发 SELL→UNDERWEIGHT 降档。LLM 误判的根源是 prompt 没把"≤"和"<"的差别突出，LLM 默认按"边界仍属原档位"的常识处理。

### 1.2 漏洞 B：压力测试方法论模糊

原 spec 写："找出你打分中**最有说服力的 2 条论据**（无论多头还是空头）"。

实际 5 只标的中：
- 300857（UNDERWEIGHT 倾向）：选了空头最强 2 条 → 自检有效，d 从 -1 移到 0
- 159326（SELL 倾向）：选了空头最强 2 条 → 自检有效，d 从 -2 移到 -1
- 02616.HK（SELL 倾向）：选了多头 1 条 + 空头 1 条 → **两边各弱 1 分相互抵消，d 从 -1.8 不变 -1.8，自检失效**

证据 —— `reports/02616.HK_CSTONE PHARMA-B_20260509_223946/2_research/manager.md` 第 67-69 行：

```
| 原始 | 24 (4.8均) | 33 (6.6均) | -1.8 |
| 弱化后（各-1分） | 22 (4.4均) | 31 (6.2均) | -1.8 |
```

LLM 选择的方法论虽然字面上符合 spec，但完全失去了压力测试的意义——**两边各弱化等量论据，差值数学上必然不变**。

正确的金融逻辑：压力测试应该质疑**当前结论的支撑面**。SELL 的支撑面是空头论据，所以要压空头最强 2 条来检验"如果我的看空依据被打折扣，结论还成立吗"。压多头论据反而是在加强 SELL 的相对优势，方向反了。

## 4. 详细变更

### 4.1 阈值表的边界表述

把现有阈值表（research_manager.py 第 54-60 行附近）：

```
| 得分差 d | 评级 |
| d > 1 | BUY |
| 0.5 ≤ d ≤ 1 | OVERWEIGHT |
| -0.5 < d < 0.5 | HOLD |
| -1 ≤ d ≤ -0.5 | UNDERWEIGHT |
| d < -1 | SELL |
```

改为以下表述（**用文字+示例消除边界歧义**）：

```
| 得分差 d | 评级 | 区间端点说明 |
|---------|------|--------------|
| d > 1（严格大于）| BUY | d=1.01 是 BUY，d=1.0 不是 |
| 0.5 ≤ d ≤ 1 | OVERWEIGHT | d=0.5、d=1.0 都是 OVERWEIGHT（双闭区间）|
| -0.5 < d < 0.5（严格不等）| HOLD | d=0.5、d=-0.5 都不是 HOLD |
| -1 ≤ d ≤ -0.5 | UNDERWEIGHT | d=-0.5、d=-1.0 都是 UNDERWEIGHT（双闭区间）|
| d < -1（严格小于）| SELL | d=-1.01 是 SELL，d=-1.0 不是 |
```

并在表格下方加一句**强制澄清**：

> **重要**：边界值（d = ±0.5、±1.0）按上表归属，不要按"还在更激进档位"的直觉处理。例如 d = -1.0 必定属于 UNDERWEIGHT，不属于 SELL；d = 1.0 必定属于 OVERWEIGHT，不属于 BUY。

### 4.2 敏感性自检的方法论澄清

把现有"第二步 B：敏感性自检"小节的第 1-2 步：

```
1. 找出你打分中**最有说服力的 2 条论据**（无论多头还是空头）
2. 假设这 2 条论据各自被弱化 1 分（例如从 7 降到 6），重新计算得分差
```

改为：

```
1. 识别你**当前评级方向的支撑论据**：
   - 若初步评级为 BUY 或 OVERWEIGHT：找多头最强 2 条论据
   - 若初步评级为 SELL 或 UNDERWEIGHT：找空头最强 2 条论据
   - 若初步评级为 HOLD：跳过自检（HOLD 不需要降档）
2. 假设这 2 条**支撑论据**各自被弱化 1 分，重新计算得分差
   - 注意：只弱化支撑当前方向的论据，**不要**两边各压一条
   - 这是压力测试的标准方法论："如果我的核心依据被打折扣，结论还成立吗？"
```

并在小节末尾保留原有的判断与降档规则（第 3-5 步），但在第 3 步前加一句：

> 在判断"是否跨过相邻档位边界"时，**严格按照阈值表的端点归属**（参见第二步阈值表的"区间端点说明"列）。例如原始 d=-2 落在 SELL 区间，弱化后 d=-1 应判为已进入 UNDERWEIGHT 区间（跨档），而非"仍在 SELL"。

### 4.3 不需要改的部分

- 输出"敏感性自检"小节的格式要求
- 第三步的行动方案表

## 7. 风险与回滚

### 7.1 主要风险

- **LLM 仍按直觉判边界**：即便 prompt 写明，LLM 在边缘 case 上仍可能按训练分布判读。需在测试时人工抽查 159326 一份
- **方法论改动让 SELL/BUY 评级数变多**：因为原本错误的"两边各压"会让自检容易"通过"（d 不变），改成"只压同侧"后，BUY/SELL 评级会更频繁触发降档到 OVERWEIGHT/UNDERWEIGHT。这是设计预期效果，不是 bug

### 7.2 回滚

- 单文件 commit。建议 message：`fix(rating): 敏感性自检边界包含与压力测试方向修正`
- 出问题 `git revert` 即可
