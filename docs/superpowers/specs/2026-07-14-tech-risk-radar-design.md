# A 股盘前与盘中科技风险雷达设计

## 背景

现有 `tradingagents.harness.market_risk_daily` 能在开盘前根据沪深 300、创业板和科创 50 的趋势、波动率与宽度代理给出全市场风险。2026-07-13 08:30 的快照已经给出“高风险、偏空、WAIT、新增仓位上限 3%”，但它无法明确指出风险集中在科技板块，也没有盘中更新和科技股专项门控。

本功能不承诺预测单日涨跌，而是提前识别风险积累、开盘后风险扩散和尾盘风险延续，并把结果转化为可执行的买卖约束。

## 目标

1. 每个 A 股交易日盘前生成“大盘风险 + 科技风险”双层快照。
2. 在 09:35、11:15、14:30 更新盘中快照。
3. 09:35 固定推送；11:15、14:30 仅在风险或操作建议变化时推送。
4. 科技风险自动联动 AI、半导体、CPO、算力等科技股票的短期入场门控。
5. 长期评级与短期风险继续解耦，科技风险不得直接修改 BUY/OVERWEIGHT/HOLD 等评级。
6. 所有快照保存数据时间与推送状态，数据不足不得伪装成低风险。

## 不在范围

- 不建设秒级行情流或自动交易系统。
- 不根据单次盘中波动自动修改长期目标价。
- 不假设系统知道用户实际持仓；`REDUCE` 仅作为已持仓场景建议。
- 不使用未来数据补写历史快照。

## 架构

### 双层风险

`market_risk` 保留现有全市场风险。新增 `tech_risk`，独立计算科技板块风险，再由 `effective_gate` 合成当前操作约束。

科技股票：

- 全市场风险高或极高：所有股票强制 `WAIT`。
- 科技风险高：科技股票禁止新开仓，当前动作降为 `WAIT`，新建仓位 0%。
- 科技风险极高：科技股票新建仓位 0%，已持仓场景建议 `REDUCE`。
- 风险恢复：只解除风险闸门，个股仍需通过自身结构与基本面门控。

### 运行时点

| 时点 | 类型 | 推送策略 |
|---|---|---|
| 08:30 | 盘前 | 固定推送 |
| 09:35 | 盘中开盘确认 | 固定推送 |
| 11:15 | 上午趋势确认 | 仅状态或建议变化时推送 |
| 14:30 | 尾盘风险确认 | 仅状态或建议变化时推送 |

非交易日不运行。同一市场、日期、时点重复执行采用 upsert，且推送使用幂等键。

## 数据输入

### 盘前

- 大盘代理：沪深 300、创业板、科创 50。
- 科技篮子：科创 50、创业板、半导体、通信、人工智能等可配置 ETF。
- 隔夜代理：纳斯达克、半导体指数及可用的中概科技指数。
- 每个输入必须携带实际数据日期；超过允许时效的输入降级为缺失。

### 盘中

- ETF 最新价、昨收、涨跌幅、成交额。
- 科技篮子同步下跌比例。
- 科技篮子中位涨跌幅。
- 科技相对沪深 300 的强弱。
- 成交额相对同一时刻基准的放大程度；基准不可得时不计分。

实时源优先；实时源失败时输出 `data_status=partial/insufficient`，不得使用旧收盘价冒充盘中实时价格。

## 科技风险评分

评分必须由 Python 确定性计算，不交给 LLM。

### 盘前因子

- 科技趋势：科技篮子低于 MA20/MA50 的比例。
- 科技宽度：科技篮子处于 MA20 上方的比例。
- 相对强弱：科技篮子相对沪深 300 的 5 日与 20 日收益差。
- 隔夜风险：海外科技代理的同步下跌与波动。
- 波动风险：科技篮子已实现波动率。

### 盘中因子

- 同步下跌比例。
- 中位涨跌幅。
- 相对沪深 300 的超额跌幅。
- 成交放大或放量下跌。
- 风险扩散：大盘与科技同时恶化。

每项因子输出 0-2 分和可读原因。总分映射为低、中、高、极高，并保留版本号，后续由 harness 按版本校准阈值。

## 状态变化与推送

比较当前时点与同日上一个时点：

- 大盘风险等级变化；
- 科技风险等级变化；
- `effective_gate` 或建议动作变化；
- 数据状态从 fresh 变为 insufficient。

09:35 无条件推送。11:15、14:30 仅上述任一项变化时推送。推送失败不阻止快照落库，但必须记录 `push_status=failed` 和错误摘要。

## 飞书报告格式

```text
【科技风险预警 09:35】

大盘风险：高 ↑
科技风险：极高 ↑
科技宽度：82% 下跌
科技中位涨跌：-2.4%
相对沪深300：-1.8%

当前动作：停止新开科技仓；已持仓高 Beta 科技股考虑降低风险敞口
新增科技仓位上限：0%
下次检查：11:15

主要原因：科技同步下跌、相对强弱恶化、下跌放量
数据状态：fresh｜数据时间：09:35
```

## 报告阅读顺序

用户只需依次看四处：

1. **当前动作**：决定现在是否允许买入或是否要降低风险。
2. **大盘风险 / 科技风险**：判断风险是系统性的，还是集中在科技板块。
3. **风险变化箭头**：`↑` 代表比上一时点恶化，`↓` 代表改善，`→` 代表不变。
4. **主要原因与数据状态**：确认是宽度、相对强弱、放量下跌中的哪一项触发；`partial/insufficient` 时降低对结论的信任。

操作解释：

| 科技风险 | 空仓者 | 科技持仓者 |
|---|---|---|
| 低 | 按个股信号执行 | 正常管理 |
| 中 | 仅小仓或等确认 | 不追涨，收紧止损 |
| 高 | 不新开科技仓 | 控制仓位，等待风险恢复 |
| 极高 | 新建仓位 0% | 建议降低高 Beta 科技风险敞口 |

最终仍以个股 `decision.md` 顶部“短期操作结论”为执行入口；风险雷达提供市场级上限，不替代个股基本面判断。

## 存储

新增盘中快照表或扩展现有表，唯一键至少包含：

- `market`
- `as_of_date`
- `session_slot`（preopen/0935/1115/1430）
- `snapshot_version`

保存大盘风险、科技风险、因子 JSON、操作建议、数据状态、数据时间、推送状态和错误摘要。

## 测试与验收

1. 盘前高科技风险正确禁止科技股新开仓，但不修改长期评级。
2. 非科技股票只受全市场风险影响，不被科技风险误伤。
3. 09:35 固定推送；11:15、14:30 无变化时不推送。
4. 风险升级、恢复和数据失效均触发变化推送。
5. 实时源失败时不得使用旧数据生成低风险。
6. 同一时点重跑不产生重复记录或重复推送。
7. 历史市场风险测试、买入时机测试和 harness 归档测试全部通过。
8. 使用 2026-07-13 场景回放，至少识别为大盘高风险；科技篮子数据满足条件时识别科技高或极高风险。
