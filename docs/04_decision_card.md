# 优化 04：决策卡（Decision Card）标准化

## 1. 背景与现状问题

### 1.1 用户痛点的源头

用户最早提出："看 reports 下对应股票的 5_portfolio 报告就够了？" 我评审后发现：**够看终局结论，但不够快**。当前 `5_portfolio/decision.md` 没有标准化的摘要头，读者必须从头读 200-400 行才能从散落的章节中拼凑出关键决策点（评级、目标价、止损、仓位、入场时机、催化、风险）。

### 1.2 现状结构

读 `reports/300308_中际旭创_20260510_153422/5_portfolio/decision.md`（典型 BUY 报告）和 `reports/AAPL_Apple Inc._20260510_114813/5_portfolio/decision.md`（典型 HOLD 报告），结构都是：

1. 一、决策前提 / 决策概览
2. 二、方向判断
3. 三、评级确定
4. 四、执行摘要（含表格但字段不固定）
5. 五、投资论点（详述 5-10 个论据）
6. 六、风险提示
7. 七、最终结论

**问题**：
- "执行摘要"小节里的表格在不同报告中字段不一致——300308 有 9 行执行要素表（含止损/目标价/仓位等），AAPL 的执行摘要更简略
- 关键数字（目标价、止损位、入场时机）散落在多个章节，没有统一的"卡式"摘要让读者一眼抓到
- 入场时机更是被埋在长段文字里，常常以"逢低吸纳""可考虑分批介入"等模糊表述出现

### 1.3 行业基准

专业卖方研报第一页（或文首）必有标准化"评级摘要卡"：
- 高盛、大摩、中金的研报都遵循统一模板
- 字段固定：Rating / Target / Time Horizon / Position / Catalysts / Risks
- 读者不需读全文就能把握决策

当前框架完全不具备这层标准化输出。

---

## 2. 目标

完成后达到：每一份 `decision.md` 文件**第一行内容**（在标题之后、任何其他章节之前）都是一张固定 9 字段的"决策卡"表格，9 个字段全部填写、无"待评估"等占位符。

读者扫一眼这张表就能拿到 PM 的核心结论；之后的详述章节是"展开论证"，不是"摘要的替代品"。

## 4. 详细变更

### 4.1 在 prompt 中插入"第零步：决策卡（强制）"

在 `portfolio_manager.py` 现有的"第三步：输出报告"指令**之前**，插入新的"第零步：决策卡"段落，要求 PM 在报告输出时把决策卡放在**所有其他章节之前**（在文档标题如 "# 002460（赣锋锂业）投资组合经理最终决策报告" 之后即出现）。

#### 决策卡的强制格式

要求 PM 输出如下表格（**字段名固定，顺序固定**）：

```markdown
## 决策卡

| 字段 | 内容 |
|------|------|
| 评级 | <BUY / OVERWEIGHT / HOLD / UNDERWEIGHT / SELL> |
| 评级置信度 | <高 / 中 / 低> |
| 目标价（区间）| <例：300-330 元> |
| 止损位 | <例：247 元> |
| 时间窗 | <短期(1-3 月) / 中期(3-12 月) / 长期(>12 月)> |
| 建议仓位 | <例：≤20% / 30-50% / 50-100%> |
| 入场时机 | <立即建仓 / 等回调到 X 元 / 等突破 Y 元 / 不建议介入> |
| 核心催化 | 1. <≤30 字> 2. <≤30 字> 3. <≤30 字> |
| 核心风险 | 1. <≤30 字> 2. <≤30 字> 3. <≤30 字> |

---
```

### 4.2 字段衍生规则（必须在 prompt 中讲清楚）

每个字段的填写来源和约束：

#### 4.2.1 评级
- 来源：你在第二步做出的最终评级（已采纳或调整 RM 评级后）
- 取值：5 档之一，与第三步报告里的"评级"一致

#### 4.2.2 评级置信度
- 来源：基于 Research Manager 给出的得分差 |d| 绝对值
- 映射规则（在 prompt 里给出明确表）：

| RM 得分差 \|d\| | 置信度 |
|----------------|--------|
| > 1.5 | 高 |
| 1.0 - 1.5 | 中 |
| < 1.0 | 低 |

#### 4.2.3 目标价（区间）
- 来源：从 Research Manager 的投资计划（投资论点中的目标价）+ 风控辩论中提到的具体目标位
- 必须是**具体数字 + 单位**（A 股用元、港股用港元、美股用美元、ETF 用元）
- 允许给区间（例：300-330 元）
- **禁止**："灵活调整"、"待评估"、"短期看涨"等模糊表述

#### 4.2.4 止损位
- 来源：从 Trader 执行方案中的"止损位评估"小节直接采用，可微调（参考 PM 在执行可行性核查中的判断）
- 必须是具体数字 + 单位
- HOLD/UNDERWEIGHT/SELL 评级也需填止损位（用于现有持仓者的风控）
- **禁止**：模糊表述

#### 4.2.5 时间窗
- 三选一：短期(1-3 月) / 中期(3-12 月) / 长期(>12 月)
- 选择依据：催化兑现的预期窗口、技术形态有效期、基本面拐点出现时间

#### 4.2.6 建议仓位
- 必须包含具体百分比（区间或上限）
- 按评级匹配（沿用优化 02 已确立的标准）：
  - BUY：50-100%
  - OVERWEIGHT：≤30%（保留 ≥50% 现金）
  - HOLD：维持现有（如已持有，不超过 20%）
  - UNDERWEIGHT：减仓 30-50%
  - SELL：清仓或 ≤10%
- 若 PM 在第二步因执行风险调整了评级（沿用优化 02 的 ±1 档调整），仓位也对应调整

#### 4.2.7 入场时机
- 4 选 1：
  1. **立即建仓**：当前价位适合直接建仓
  2. **等回调到 X 元**：明确给具体回调目标价
  3. **等突破 Y 元**：明确给具体突破确认价
  4. **不建议介入**：当前不是合适入场时点（适用于 HOLD/UNDERWEIGHT/SELL，或 BUY 但当前价位偏高）
- 选项 2/3 必须给具体数字
- 本次 spec 不要求 PM 在 prompt 里有专门的"入场时机决策算法"——PM 凭对当前价位 vs 目标价/支撑位的常识判断填即可。**正式的入场时机决策逻辑放在优化 05**

#### 4.2.8 核心催化
- 最多 3 条 bullet
- 每条 ≤30 字
- 来源：从 RM 投资计划 / bull 论据 / news 报告中提炼最关键的 3 个上行催化
- 例：
  - "Q2 净利润 >10 亿确认业绩持续"
  - "国常会强调算力网建设"
  - "光芯片量产+50% 毛利率"

#### 4.2.9 核心风险
- 最多 3 条 bullet
- 每条 ≤30 字
- 来源：从 RM 投资计划 / bear 论据 / 风控辩论中提炼最关键的 3 个下行风险
- 例：
  - "融资环境收紧触发流动性危机"
  - "Q2 业绩低于 5 亿引发戴维斯双杀"
  - "算力价格反转 30%+"

### 4.3 决策卡之后保留现有详述

明确告诉 PM：决策卡之后必须保留现有的所有详述章节（投资论点、执行摘要、风险提示等）。**决策卡是摘要，不是替代**——读者要快速决策时看卡，要核对推理时读详述。

prompt 中可以这样表述：

> 决策卡是给读者的"封面摘要"，目的是 30 秒内传达决策。卡之后的详述章节（投资论点、执行摘要、风险提示）必须保留，不得删减。读者审议时会读详述，但行动时只看卡。

### 4.4 不需要改的部分

- 第一步：方向判断
- 第二步：PM 评级决策规则（默认采纳 + ±1 档调整 + 不跨方向翻转，已在优化 02 确立）
- 执行可行性核查（已在优化 01 加入）
- 历史教训和风控辩论的注入逻辑

## 7. 风险与回滚

### 7.1 主要风险

- **PM 用模糊表述应付**：LLM 可能写"目标价：参考机构目标价区间"或"止损位：见详述"，规避具体数字。验收 5.2 的"人工抽查"检测此模式
- **核心催化/风险写超字数**：30 字限制可能被忽略。验收时人工抽查
- **决策卡和详述出现矛盾**：例如卡里写"立即建仓"，但详述里写"等回调"。需在 prompt 里强调"卡是详述的摘要，必须一致"
- **删减详述**：PM 可能为了简化而把详述章节删了。验收 5.1 最后一项检测

### 7.2 回滚

- 单文件 commit。建议 message：`feat(pm): decision.md 头部强制 9 字段决策卡`
- 出问题 `git revert` 即可
