## 4. "至少 2 只 OVERWEIGHT/UNDERWEIGHT" 未通过的讨论

### 现象

5 只标的中，0 只最终产出 OVERWEIGHT/UNDERWEIGHT。但 **300857 在中间步骤产出了 UNDERWEIGHT**（d=-1，恰好落在 -1 ≤ d ≤ -0.5 区间），经敏感性自检后降档为 HOLD。

### 分析

1. **5 档阈值机制确实生效**：300857 的得分差 d=-1 按新阈值被判定为 UNDERWEIGHT（旧版二档制下 d≤1 全部为 HOLD）。这说明阈值放松确实让中间档位得以产出。

2. **敏感性自检正在做它该做的事**：300857 的空头最强2条论据（现金流背离8分、负债率飙升7分）各弱化1分后，空头得分从7降至6，得分差从-1变为0，跨过了 UNDERWEIGHT→HOLD 的边界。这说明原始 UNDERWEIGHT 评级确实是脆弱的——建立在 LLM 评分的边缘扰动上。自检正确地将其降档。

3. **LLM 评分倾向极端化**：观察 5 只标的的得分差分布（-2, -1.8, -1, -0.4, 0.4），LLM 倾向于给出明显偏向某一方的评分，而非谨慎的中间值。这意味着得分差落在 [0.5, 1] 或 [-1, -0.5] 区间的概率本身较低。

### 结论

5 档评级机制和敏感性自检均按设计工作。OVERWEIGHT/UNDERWEIGHT 未在最终评级中出现，并非因为 prompt 未生效，而是因为：
- 本次跑批中，多空得分差要么足够大（明确 SELL），要么足够小（均衡 HOLD），或处于边缘被自检降档
- 敏感性自检正确拦截了1个脆弱的 UNDERWEIGHT

如果需要强制产出 OVERWEIGHT/UNDERWEIGHT 用于验收，有两个选项：
1. **多跑几只标的**：LLM 评分有随机性，得分差落在 [0.5, 1] 区间的标的终会出现
2. **收窄 HOLD 区间**：将 HOLD 从 -0.5 < d < 0.5 改为 -0.3 < d < 0.3，扩大 OVERWEIGHT/UNDERWEIGHT 的覆盖面

### 5.1 300857 协创数据（触发降档的案例）

```markdown
### 第二步 B：敏感性自检

| 步骤 | 内容 |
|------|------|
| **最有说服力的2条论据** | ① 空头：现金流与利润严重背离（7.50亿利润vs2.72亿现金）；② 空头：资产负债率一年内飙升20个百分点 |
| **假设弱化1分后** | ① 现金流背离论据：7→6；② 负债率飙升论据：8→7 |
| **弱化后重新计算** | Bull Score: 6（不变）；Bear Score: 7→6；差值 d = 6 - 6 = **0** |
| **是否触发降档** | d 从 -1 变为 0，跨过边界（从 UNDERWEIGHT 区进入 HOLD 区）→ **触发降档** |
| **最终评级** | HOLD（中性） |
```

## 7. 复盘总结

### 7.1 改造效果

- **5 档评级机制已生效**：RM 输出中出现了 5 档阈值表引用和对应评级判定，不再是旧版的"得分差>1→Buy/≤1→Hold"二档硬切
- **敏感性自检已常态化**：所有非 HOLD 评级（3 只）均包含完整的敏感性自检小节，其中 1 只（300857）正确触发降档
- **PM 与 RM 解耦但克制**：5 只标的 PM 均默认采纳 RM 评级，无跨方向翻转，无不当调整
- **无 STRONG_BUY/STRONG_SELL 泄漏**：5 份 manager.md 均严格产出 5 档评级

### 7.2 发现的问题

1. **OVERWEIGHT/UNDERWEIGHT 产出率低**：本次 5 只标的 0 只最终落在中间档位。300857 曾落入 UNDERWEIGHT 但被自检降档。LLM 评分倾向极端化是根本原因，可能需要通过收窄 HOLD 区间或多跑标的解决。

2. **得分差边界判定有歧义**：300857 的 d=-1 恰好落在 SELL 区间的边界（d < -1 vs -1 ≤ d ≤ -0.5）。LLM 在自检过程中出现了"等等，让我再核实一下"的自我纠错，说明阈值表述在边界处不够清晰。建议将阈值表改为左闭右开/左开右闭的明确表述。

3. **敏感性自检的计算方式不一致**：300857 的自检是将空头2条论据同时弱化（bear: 7→6），导致 d 从 -1 变为 0；而 AAPL 的自检是分别弱化多头/空头各1条，产生两个场景。规格书原文说"最有说服力的2条论据各弱化1分"，应统一为：不论多空，选最强2条同时弱化。

### 7.3 与优化 01 的交互验证

优化 01 的 Trader 折叠规则（OVERWEIGHT→BUY、UNDERWEIGHT→SELL）本次无实例触发，但规则仍在 trader.py 中完好保留，无回归风险。
