# 优化 04b：决策卡补丁——置信度映射 + 入场时机 + 催化/风险格式

## 1. 背景

优化 04 验收（[中期状态文档](04_decision_card_中期状态.md)）暴露 3 处硬伤，**结构层面 100% 通过**，但**字段填写层面**有 LLM 不严格遵守 prompt 的问题：

### 1.1 硬伤 A：置信度映射 3/6 错误（最严重）

证据 ——
- 300857: `|d|=0.76 < 1.0`，应填"低"，实填"中"
- 02616.HK: `|d|=0.80 < 1.0`，应填"低"，实填"中"  
- AAPL: `|d|=1.42`（落在 1.0-1.5），应填"中"，实填"高"，且 LLM 自己写"\|d\|=1.42 > 1.5"——**数学幻觉**

原 spec 已给出明确映射表，但 LLM 在数值边界附近（特别是 0.76、0.80 这种"接近 1"的情况）会凭直觉判"中"而非严格按区间选"低"。AAPL 案例更是 LLM 直接写错了不等式。

### 1.2 硬伤 B：入场时机 4 选 1 被混合（1/6 违例）

证据 —— 300308 的入场时机字段：

```
等回调到 750-800 元建第一批；等突破 900 元确认后建第二批
```

这把 spec 的"等回调到 X"和"等突破 Y"两个**互斥选项**塞在一起。spec 要求 4 选 1。

### 1.3 硬伤 C：核心催化/风险格式不一致（1/6 + 部分超字数）

证据 ——
- 159326 把每条催化拆成独立行（3 个"核心催化"标签独立出现），破坏单行表格结构
- 部分标的单条 >30 字（300857 的"债务风险：资产负债率85.89%，流动比率0.90"约 25-30 字边缘）

---

## 2. 目标

通过 prompt 层精细化约束，把这 3 处硬伤的命中率拉到：

1. **置信度映射**：6/6 全对（必须先复算 \|d\| 数值，再严格对照区间表）
2. **入场时机**：6/6 严格 4 选 1，不允许混合多个选项
3. **核心催化/风险**：6/6 单行内编号、单条 ≤30 字

## 4. 详细变更

### 4.1 置信度映射强化

把现有第 2 项"评级置信度"段落（portfolio_manager.py 第 87-93 行附近）从：

```
2. **评级置信度**：基于 Research Manager 给出的得分差 |d| 绝对值

   | RM 得分差 |d| | 置信度 |
   |----------------|--------|
   | > 1.5 | 高 |
   | 1.0 - 1.5 | 中 |
   | < 1.0 | 低 |
```

改为：

```
2. **评级置信度**：基于 Research Manager 给出的得分差 |d| 绝对值

   **执行步骤（强制按顺序）**：
   
   步骤一：从 Research Manager 的报告中提取得分差 d 的数值（保留 2 位小数）
   步骤二：计算 |d|，**显式写出数值**（不允许凭印象估计）
   步骤三：严格按下表选择置信度，**只看数值是否落在区间内，不允许凭直觉判断**：

   | |d| 范围 | 置信度 | 端点归属 |
   |---------|--------|---------|
   | |d| > 1.5（严格大于）| 高 | |d|=1.51 是高，|d|=1.50 不是 |
   | 1.0 ≤ |d| ≤ 1.5 | 中 | |d|=1.00、|d|=1.50 都是中 |
   | |d| < 1.0（严格小于）| 低 | |d|=0.99 是低，|d|=1.00 不是 |

   **常见错误（必须避免）**：
   - ❌ |d|=0.76 但填"中"——错误！0.76 < 1.0 必须填"低"
   - ❌ |d|=1.42 但填"高"且写"|d|=1.42 > 1.5"——错误！1.42 < 1.5，是"中"
   - ✅ |d|=1.42 → 1.0 ≤ 1.42 ≤ 1.5 → "中"
   
   **必须**在决策卡的"评级置信度"字段同时写出 |d| 的具体数值，便于读者复核。
   例如：`| 评级置信度 | 低（|d|=0.76）|`
```

### 4.2 入场时机收紧

把现有第 7 项"入场时机"段落改为：

```
7. **入场时机**：严格四选一，**只能选一个**：
   
   - 选项 A：**立即建仓**（当前价位适合直接介入）
   - 选项 B：**等回调到 X 元**（明确给具体回调目标价）
   - 选项 C：**等突破 Y 元**（明确给具体突破确认价）
   - 选项 D：**不建议介入**（适用于 HOLD/UNDERWEIGHT/SELL 或当前价位偏离严重）
   
   **关键约束**：
   - **禁止混合两个选项**——不允许"等回调到 X，再等突破 Y"这种写法
   - 如果你的策略是"分批建仓"，选 B（取第一批建仓的回调位作为 X）或 C（取突破确认位作为 Y），但只能选一个
   - 允许在选定的选项后**附加 ≤20 字**的补充说明（如"不建议介入（持仓者逢高减仓）"），但不能引入另一个选项
   
   **错误示例**：
   - ❌ "等回调到 750-800 元建第一批；等突破 900 元确认后建第二批"——混合了 B 和 C
   - ✅ "等回调到 750-800 元"（只选 B）
   - ✅ "等突破 900 元"（只选 C）
```

### 4.3 核心催化/风险格式

把现有第 8、9 项改为：

```
8. **核心催化**：最多 3 条，单行内编号格式：`1. xxx 2. xxx 3. xxx`
   - 每条 **≤30 字**（含标点和数字）
   - 来源：RM 投资计划 / bull 论据 / news 报告中提炼最关键的上行催化
   - **格式约束**：必须在一行内用编号 "1. 2. 3." 分隔，**禁止换行成多个独立行**
   
   **正确示例**：
   ```
   | 核心催化 | 1. Q2 业绩超预期增 80% 2. 政策落地资金注入 3. 行业景气 ETF 净流入 |
   ```
   
   **错误示例**（禁止）：
   ```
   | 核心催化 | 1. xxx |
   | 核心催化 | 2. xxx |
   | 核心催化 | 3. xxx |
   ```

9. **核心风险**：最多 3 条，**与"核心催化"相同的格式约束**——单行内编号、每条 ≤30 字、禁止多行展开

   **写每条时的字数控制技巧**：用"主谓宾"短句，避免冒号展开。
   - ❌ "债务风险：资产负债率85.89%，流动比率0.90"（30字以上，含冒号展开）
   - ✅ "负债率 86% 叠加流动比率 0.90 偿债压力大"（24 字）
```

### 4.4 不需要改的部分

- 决策卡的整体格式表（9 字段、字段名、顺序）
- 字段 1（评级）、3（目标价）、4（止损位）、5（时间窗）、6（建议仓位）的填写规则
- "关键约束"小节（9 字段禁止占位符、决策卡是摘要、卡与详述一致）
- 第一步、第二步、第三步的所有其他内容

## 7. 风险与回滚

### 7.1 风险

- **LLM 仍不严格遵守**：即便加了反例和约束，LLM 在边界 case 上仍可能凭直觉判断。验收 5.1 的"6/6 全对"是硬指标——任何 1 个错都需排查并加强 prompt
- **附加文字仍被滥用**：允许 ≤20 字补充说明的口子可能被 LLM 扩大。验收 5.2 的人工抽查要把"≤20 字"算在心里

### 7.2 回滚

单文件 commit。建议 message：`fix(decision_card): 置信度映射 + 入场时机 + 催化风险格式三处强化约束`

出问题 `git revert` 即可。
